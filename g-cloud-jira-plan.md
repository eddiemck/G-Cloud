# G-Cloud 15 Readiness — Jira Plan (for review)

Drafted for the **Crozier Scott** Jira project (key `CROZ`, board KNBN). Nothing has been created in Jira yet — this is for Eddie/Scott to review, amend, and approve first.

Structure used, matching how the team organises work here (not the Jira-standard hierarchy, but workable within it):

- **Epic** — the overall project
- **Story** — each main deliverable
- **Task** — created as a **Sub-task** under its Story (Jira doesn't let a plain Task nest under a Story, so Sub-task is the type that actually achieves "tasks within stories")

Once approved, each `- [ ]` line below becomes one Jira issue.

## EPIC: G-Cloud 15 Readiness — CrozierScott

**Description:** Prepare CrozierScott to apply the moment the G-Cloud 15 framework reopens to new suppliers (expected ~March 2028). Covers building a compliant service catalogue, SFIA-based pricing, certifications, policies, case studies, technical evidence, and a reusable bid library — plus scoping DOS7 as a nearer-term alternative for bespoke work. Internal target: **fully ready by Friday 3 December 2027.**
